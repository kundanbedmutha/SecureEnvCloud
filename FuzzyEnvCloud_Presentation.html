Problem Statement — Why Did We Build This?

03 / 11
💸
Gap 1 — Cost Inefficiency
Traditional always-on VM servers waste money even when no data is coming in. We need a pay-per-use serverless architecture that scales automatically.
🔓
Gap 2 — No Security
Most IoT dashboards have zero access control — anyone can view or modify data. Real systems need proper user roles and identity management.
⚠️
Gap 3 — Reactive Alerts Only
Existing systems only alert AFTER danger arrives. We want to predict future readings and warn BEFORE hazardous conditions actually occur.

Admin Panel & Data Export

07 / 11
Admin Panel — /admin
➕Add new users with a role
🗑️Delete existing users instantly
🔄Switch roles live — no server restart needed
💾Changes saved to users.json instantly
CSV Data Export
📊/export/csv — all sensor readings
🚨/export/alerts/csv — alerts log
📋Fields: ID, Timestamp, Temp, AQI, Risk Score
✅Available to both Admin and Viewer roles
Why This Solves Gap 2 — Security as Afterthought
Before: Any user could see or modify all data — no access control whatsoever
After: Admin routes protected by decorators, viewer gets read-only access across all APIs
Real World: This is exactly how AWS IAM works — roles, policies, and dynamic provisioning
